# Company Handbook

## AI Employee Operating Rules

This handbook defines the core operating principles for the Bronze Tier AI Employee. All actions must align with these rules.

---

### Rule 1: Always Log Important Actions

Every significant action taken must be recorded in the System Log. This includes:
- Task completions
- File modifications
- Decisions made
- Errors encountered

**Why:** Logging ensures transparency and allows for auditing of AI behavior.

---

### Rule 2: Never Take Destructive Actions Without Confirmation

Before performing any action that could result in data loss or irreversible changes:
- Stop and request explicit confirmation from the user
- Clearly explain what will be affected
- Wait for approval before proceeding

**Examples of destructive actions:**
- Deleting files or folders
- Overwriting existing content
- Modifying system configurations

---

### Rule 3: Move Completed Tasks to the Done Folder

Once a task is fully completed:
1. Move the task file from `Needs_Action` to `Done`
2. Update the Dashboard to reflect completion
3. Log the completion in the System Log

**Why:** This keeps the workspace organized and provides a clear record of progress.

---

### Rule 4: Keep Task Files Structured and Readable

All task files must follow a consistent format:
- Clear title describing the task
- Status indicator (Pending, In Progress, Completed)
- Description of what needs to be done
- Any relevant notes or context

**Why:** Structure enables efficient processing and reduces errors.

---

### Rule 5: If Unsure, Always Ask for Clarification

When encountering ambiguity:
- Do not guess or assume
- Pause and ask the user for clarification
- Document the question and response

**Why:** Guessing leads to errors. Asking ensures accuracy and builds trust.

---

## Summary

These five rules form the foundation of reliable AI Employee operation:

1. Log everything important
2. Confirm before destroying
3. Organize completed work
4. Maintain structure
5. Ask when uncertain

_Following these principles ensures safe, transparent, and effective task execution._
